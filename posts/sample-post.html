개발 환경을 구축할 때 가장 먼저 고민하게 되는 것이 바로 에디터 선택이다. 수많은 선택지가 있지만, VS Code는 그 중에서도 가장 많은 개발자들이 선택하는 에디터로 자리잡았다. 오늘은 내가 직접 사용해보고 추천하는 VS Code 확장 프로그램들을 소개하려 한다.

왜 VS Code인가?

VS Code의 가장 큰 장점은 바로 풍부한 확장 프로그램 생태계다. 마이크로소프트가 오픈소스로 개발하고 있으며, 전 세계 개발자들이 자신만의 확장 프로그램을 만들어 공유하고 있다.

좋은 도구는 개발자의 생산성을 10배 이상 높여준다.

추천 확장 프로그램

1. Prettier - Code Formatter

코드 포매팅은 협업에서 필수적인 요소다. Prettier는 자동으로 코드 스타일을 일관되게 맞춰주어 팀원들 간의 스타일 논쟁을 줄여준다. 설정 한 번으로 저장할 때마다 자동 포매팅이 적용된다. 

{
  "editor.formatOnSave": true,
  "editor.defaultFormatter": "esbenp.prettier-vscode"
}

2. GitLens

Git 작업을 하다 보면 "이 코드는 누가 언제 작성했지?"라는 궁금증이 생긴다. GitLens는 각 라인별로 커밋 정보를 보여주어 코드의 역사를 쉽게 파악할 수 있게 해준다.

3. Thunder Client

API 테스트를 위해 Postman을 따로 실행할 필요 없이, VS Code 내에서 바로 REST API를 테스트할 수 있다. 가볍고 빠른 것이 가장 큰 장점이다.

4. Auto Rename Tag

HTML/JSX 작업 시 열린 태그를 수정하면 닫는 태그도 자동으로 수정해준다. 단순하지만 굉장히 유용한 확장 프로그램이다.

5. Error Lens

에러와 경고 메시지를 해당 라인 옆에 바로 표시해준다. Problems 탭을 열어볼 필요 없이 바로 확인할 수 있어 디버깅 시간을 줄여준다.

마치며

좋은 도구를 찾는 것은 개발자로서 중요한 투자다. 위에서 소개한 확장 프로그램들을 하나씩 설치해보고, 자신의 워크플로우에 맞는 것들을 선별해서 사용하길 바란다. 생산성 향상은 작은 습관의 변화에서 시작된다.
