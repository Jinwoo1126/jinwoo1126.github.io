## AI 에이전트 스킬(Agent Skills)

### 1. 에이전트 스킬 개요

- **정의:** LLM이 외부 세계와 상호작용하고 실제 작업을 수행할 수 있도록 부여하는 **'도구(Tools)'**이자 **'실행 능력'**
    
- **역할:** 단일 텍스트 생성을 넘어 자율적인 **계획(Planning) → 검색(Retrieval) → 제어(Action)** 루프를 완성하는 핵심 요소
    

---

### 2. 에이전트 스킬의 주요 유형

|**스킬 유형**|**목적**|**주요 예시**|
|---|---|---|
|Retrieval<br><br>  <br><br>(정보 검색)|내부 지식베이스나 외부 환경에서 필요한 최신 및 전문 정보 확보|• Vector DB (OpenSearch, pgvector 등) 조회<br><br>  <br><br>• 웹 검색 API (Tavily 등) 연동<br><br>  <br><br>• 복잡한 문서 파서 (Docling 등) 실행|
|Action<br><br>  <br><br>(시스템 제어)|단순 조회를 넘어 외부 시스템의 상태(State)를 변경하거나 트랜잭션 발생|• 사내 업무 시스템 API 호출 (티켓 생성 등)<br><br>  <br><br>• DB 데이터 쓰기/수정<br><br>  <br><br>• 스마트 디바이스 제어|
|Compute<br><br>  <br><br>(연산/논리)|LLM의 약점인 환각을 방지하고 정확한 수학적/논리적 연산 수행|• Python REPL (코드 인터프리터)<br><br>  <br><br>• 계산기 도구<br><br>  <br><br>• 시계열 분석 및 시뮬레이터 연동|

### 3. 스킬의 기술적 구현: Function Calling (Tool Calling)

LLM이 스킬을 인식하고 실행하는 4단계 메커니즘:

1. **정의 (Schema Definition):** 스킬 이름, 설명, 파라미터 타입을 JSON 스키마로 LLM에 제공
    
2. **판단 (Reasoning):** LLM이 사용자 요청을 분석하여 호출할 도구와 인자(Arguments)를 결정 (JSON 반환)
    
3. **실행 (Execution):** 애플리케이션 레이어(또는 워크플로우)에서 LLM이 지정한 인자로 실제 함수 실행
    
4. **피드백 (Observation):** 실행 결과를 LLM의 컨텍스트에 추가하여 다음 행동 결정 및 최종 답변 생성
    

---

### 4. 멀티 에이전트(Multi-Agent) 환경에서의 스킬 오케스트레이션

복잡한 태스크 처리를 위한 **Supervisor/Planner 패턴** 활용:

- **역할 분산 (Worker Agents):** 스킬의 성격에 따라 에이전트 분리 (예: 검색 전용 에이전트, 코드 작성 에이전트)
    
- **상태 그래프 공유 (State Graph):** 각 에이전트의 스킬 실행 결과(Observation)를 전역 상태로 관리하며 다음 노드(Node)의 에이전트에게 전달
    
- **Human-in-the-Loop (HITL):** DB 삭제나 중요 결제 등 크리티컬한 스킬 실행 전, 그래프의 실행을 일시 중지하고 인간의 승인을 받도록 설계
    

---

### 5. 스킬 확장성 표준: MCP (Model Context Protocol)

- **개념:** 에이전트(클라이언트)와 다양한 데이터 소스/도구(서버) 간의 양방향 연결을 규격화한 개방형 표준
    
- **효과:** 새로운 데이터 소스 연결 시마다 커넥터를 새로 개발할 필요 없이, '플러그 앤 플레이' 방식으로 로컬 환경, 사내 저장소(GitHub), 메신저(Slack) 등을 에이전트 스킬로 손쉽게 통합하여 확장성 극대화
